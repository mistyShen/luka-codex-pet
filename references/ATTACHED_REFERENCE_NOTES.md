# Attached Reference Notes

来源：用户在对话中提供的巡音流歌 Codex 宠物设计板。部分脸部区域被模糊遮挡；这些遮挡区域不作为细节依据。

## 可见设计要点

- 角色是 2.5 到 3 头身 Q 版歌姬少女，整体比例明显头大身小。
- 主体发色为柔和粉色，长发覆盖后背，发尾厚实圆润，有轻微卷曲和分束。
- 头部有黑金色发箍/耳机结构，耳机外侧带蓝色点缀，金色边线清楚。
- 服装是黑色底、金色描边和装饰为主，局部有橙金色花纹，整体偏精致歌姬装但需要简化到小尺寸可读。
- 眼睛在可见区域呈紫色/蓝紫色，表情整体温柔、平静、成熟。
- 伴随一个小粉章鱼：圆润头部、短小触手、黑色小眼睛、柔和粉色高光，必须保持可爱干净，不能怪物化。

## 可见动作状态

- `idle`：角色安静停留，小粉章鱼在旁边；适合轻微呼吸、头发微摆。
- `running-right`：向右拖拽移动，长发向后滞后，章鱼跟随或贴近角色。
- `running-left`：向左拖拽移动，动作与右移相反，角色和章鱼都保持小幅运动。
- `waving`：角色正面小幅挥手，旁边章鱼同步存在；可以用角色手臂和章鱼触手表达欢迎。
- `jumping`：轻轻跳起，头发和服装上扬，小粉章鱼可轻微弹起。
- `failed`：角色靠近章鱼，章鱼可持一个红色错误色块；需要避免可读文字和漂浮红叉。
- `waiting`：角色抱着小粉章鱼，安静等待；参考图中有省略号语义，但 spritesheet 中用姿态表达，不画文字省略号。
- `running`：Codex 工作中，角色面对终端/日志类道具，小章鱼参与工作。
- `review`：角色拿结果板/报告卡片，小章鱼持确认道具；表达检查结果。
- `complete`：参考图有完成/成功状态，但 Codex 当前 9 行状态不含该行，可作为 `jumping` 或 `review` 的情绪参考。

## 可见小章鱼道具

- 敲键盘。
- 看日志。
- 抱终端窗口形小板。
- 抱进度条形道具。
- 抱报错标记。
- 持任务清单/报告卡。
- 拿麦克风。
- 拿音符道具。

## 生成约束

- 参考图可作为角色身份和动作语义来源，但不要复制可读文字、官方标识、UI 截图或复杂细节。
- 终端、日志、进度条、报告卡、错误标记都必须简化成无字小道具，并贴近角色或被角色/章鱼拿着。
- 不要漂浮音符、漂浮省略号、漂浮红叉、粒子特效、地面阴影或舞台背景。
- 小尺寸优先：黑金服装细节要保留大块轮廓和少量金色线条，不追求复杂花纹。
